# Reproducibility guide

Run all commands from the repository root. The scripts deliberately avoid
implicit session objects: each stage reads files written by an earlier stage
and writes a machine-readable result.

## Execution order

1. `python scripts/01_preprocess.py --force`
2. `pytest -q`
3. `Rscript scripts/02_mfunction_free.R`
4. `Rscript scripts/03_mfunction_county.R`
5. `Rscript scripts/04_permanova_permdisp.R`
6. `python scripts/05_topotest.py --mode both`
7. `Rscript scripts/06_ball_mapper.R`
8. `Rscript scripts/07_reconciliation.R`
9. Run the scripts in `scripts/robustness/` and `scripts/figures/` as needed.

All stochastic stages record their seed and repetition count in their output
tables or serialized arrays. M-function output also records the worker count,
because its parallel random streams depend on that setting.

## Long-run snapshots

The repository includes the final M-function null arrays because reconstructing
them takes several hours. They are not required for preprocessing tests and can
be regenerated by stages 02 and 03. Figure-backing CSV files allow the final
plots to be inspected without rerunning every long permutation stage.
